Animations

Most obviously, animations - as used on homepage and quiz.

Using keyframes, animations in CSS is actually very simple. The transformation function will be applied dynamically to the element's display, changing the element's attributes according to the pre-set parameters assigned by the designer. There are multiple parameters to adjust to nudge and tweak animation speed, element state before and after, interpolation style, play direction and looping settings. Moreover, one animation can be re-applied multiple times to many elements, so give it a try!

Animations have been applied to quiz page in a way that gives the impression of interactivity: each fieldset responds to being hovered on by expanding in width, demanding user's focus and improving accessibility while receiving input. This is done through the use of @keyframes and the pseudo-selector :hover.

Parallax

A little bit more subtle, as used on topic page and right here.

The idea behind a parallax scrolling effect is to shift the elements on the Z-axis, to create several layers and give the impression of depth. This can be achieved through the use of the transformation function translateZ(-px) to "shift" the elements further away from the user, then apply another scale() to size it appropriately according to the law of perspective.

What are the steps to do it?

  1. Give the parent container the {perspective: -px;} attribute. This is so that all children of that element would be put in a 3D-simulated environment rather than the traditional plane of the viewport.
  2. Seperate the HTML into multiple segments, each within a <section> with either the class "parallax" or "no-parallax". Give all of them the attribute {transform-style: preserve-3d;}
  3. Apply to the pseudo selector ::after the attributes {position: absolute; top: 0; right: 0; left: 0; bottom: 0;}. Apply the transformation functions above as well. Finally, give it a different Z-index to separate it to a different layer than the base.
  4. If the elements scale up bigger than the viewport, scrollbars will appear by default. To prevent this, add the attributes {overflow-x: hidden} to the container.

Accessibility

Look up! Right on the navigation menu, the darkmode toggler is present and functional on all pages.

It is a bit more tricky to implement this without JavaScript, but it is indeed possible to accomplish with pure HTML/CSS. By using the "from...to" selector (~), and by using a hidden checkbox with the pseudo-selector ":checked", we can select a scope of the entire body and make changes to the variables within this scope. By storing all changeable color values as variables, this simple switch can implement a drastic style change across the entire page with a simple click.

Morais, R. Dark Mode without JavaScript.
https://codepen.io/rafaelsnts/pen/BEzZoX

But that's not all! Try toggling darkmode. Do the background images appear to be darker/lighter?

By applying the built-in function linear-gradient(), putting the opacity to 50% and passing the image in as a parameter, CSS automatically generates a "shader" to cover over the image. Passing this entire function as a parameter to the color variable creates two corresponding shaders to help aid in text visibility in both light and darkmode.
